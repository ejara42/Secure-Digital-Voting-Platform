Supervised learning in machine learning means training a model with labeled data (inputs paired with correct outputs). It mainly includes two types: classification and regression.

🌐 What Supervised Learning Is

  • Definition: Supervised learning is a machine learning paradigm where an algorithm learns to map inputs to outputs using labeled data. Each training example consists of an input (features) and the correct output (label).

  • Goal: The model generalizes from training data to make accurate predictions on unseen data.

  • Process:

    1. Provide the algorithm with labeled examples.

    2. The model predicts outputs.

    3. Errors are measured against the true labels.

    4. The model adjusts itself to minimize errors (using optimization techniques like gradient descent).

🔑 Types of Supervised Learning Problems

TypeOutputExamplesAlgorithms Commonly Used

RegressionContinuous (numeric values)Predicting house prices, stock market values, temperature forecastingLinear Regression, Polynomial Regression, Ridge/Lasso Regression, SVR (Support Vector Regression), Neural Networks

📌 Key Characteristics

  • Labeled Data: Essential for supervised learning; without labels, the model cannot learn the mapping.

  • Training vs. Testing: Data is split into training (to learn) and testing (to evaluate performance).

  • Generalization: The ability of the model to perform well on unseen data is critical.

  • Evaluation Metrics:

    • Classification: Accuracy, Precision, Recall, F1-score.

    • Regression: Mean Squared Error (MSE), Mean Absolute Error (MAE), R² score.

⚖️ Advantages & Challenges

  • ✅ Advantages:

    • Produces highly accurate models if enough labeled data is available.

    • Well-suited for real-world tasks like fraud detection, medical diagnosis, and recommendation systems.

  • ❌ Challenges:

    • Requires large amounts of labeled data (expensive/time-consuming to prepare).

    • Risk of overfitting (model memorizes training data instead of generalizing).

    • Performance depends heavily on data quality.

🚀 Real-World Applications

  • Healthcare: Predicting disease outcomes from patient data.

  • Finance: Credit scoring, stock price prediction.

  • Technology: Spam filters, speech recognition, image classification.

  • Agriculture: Crop yield prediction, pest detection.

🔍 Logistic Regression in Detail

1. What It Is

  • Despite the name, it’s used for classification, not regression.

  • It predicts the probability that an input belongs to a certain class (e.g., Fraudulent vs Legitimate vote).

  • Output is between 0 and 1 using the sigmoid function.

2. The Math Behind It

  • Linear part:

z=w1x1+w2x2+⋯+wnxn+b

where:

  • xi = input features (e.g., voter age, location, device type)

  • wi = weights (learned by the model)

  • b = bias term

  • Sigmoid function:

σ(z)=11+e−z

This squashes the linear output into a probability between 0 and 1.

  • Decision rule:

    • If σ(z)≥0.5 → Class = 1 (Fraudulent)

    • If σ(z)<0.5 → Class = 0 (Legitimate)

3. Training Process

  1. Initialize weights randomly.

  2. Forward pass: Compute predictions using sigmoid.

  3. Loss function: Use Binary Cross-Entropy:

L=−1m∑i=1m[yilog⁡(y^i)+(1−yi)log⁡(1−y^i)]

where:

  • yi = true label

  • y^i = predicted probability

  • m = number of samples

  1. Backpropagation: Adjust weights using gradient descent to minimize loss.

  2. Repeat until loss is small and predictions are accurate.

🌳 Decision Trees in Detail

1. What They Are

  • A Decision Tree is a model that splits data into branches based on feature values.

  • Each internal node represents a test on a feature (e.g., "Was the vote cast online?").

  • Each branch represents the outcome of the test.

  • Each leaf node represents the final prediction (e.g., Fraudulent or Legitimate).

2. How They Work

  1. Start at the root node (all data).

  2. Choose the best feature to split the data (using criteria like Information Gain or Gini Index).

  3. Split the dataset into subsets based on feature values.

  4. Repeat recursively until:

    • All samples in a node belong to the same class, or

    • No further improvement can be made.

  5. Final prediction is made at the leaf node.

3. Math Behind Splitting

  • Entropy (measure of impurity):

H(S)=−∑i=1cpilog⁡2(pi)

where pi = proportion of class i.

  • Information Gain (how much uncertainty is reduced by a split):

IG(S,A)=H(S)−∑v∈Values(A)∣Sv∣∣S∣H(Sv)

  • S = dataset

  • A = feature

  • Sv = subset after splitting by value v

  • Gini Index (alternative impurity measure):

Gini(S)=1−∑i=1cpi2

🌲 Random Forests in Detail

1. What They Are

  • A Random Forest is an ensemble method: it builds many decision trees and combines their results.

  • Each tree is trained on a random subset of the data and features.

  • Final prediction is made by:

    • Classification → Majority vote across trees.

    • Regression → Average of all tree predictions.

2. How They Work Step by Step

  1. Bootstrap Sampling: Randomly select subsets of the training data (with replacement).

  2. Feature Randomness: At each split, only consider a random subset of features.

  3. Train Multiple Trees: Each tree learns slightly different patterns.

  4. Aggregate Predictions:

    • Classification → Trees vote, majority wins.

    • Regression → Predictions are averaged.

  5. Result: More robust, less overfitting, higher accuracy.

🧠 What Is Unsupervised Learning?

  • Definition: Machine learning where the data has no labels. The algorithm must discover hidden patterns, groupings, or structures by itself.

  • Goal: Understand the underlying structure of data, reduce complexity, or detect anomalies.

  • Key Difference from Supervised Learning: No “teacher” or correct answers — the model explores and organizes data on its own.

🔑 Main Types of Unsupervised Learning

1. Clustering

  • Purpose: Group similar data points together.

  • Examples:

    • Election system: Group voters by behavior (e.g., early voters vs late voters).

    • Marketing: Customer segmentation.

    • Biology: Grouping genes with similar expression.

  • Algorithms:

    • K-Means: Divides data into k clusters by minimizing distance to cluster centers.

    • Hierarchical Clustering: Builds a tree of clusters (dendrogram).

    • DBSCAN: Finds clusters of arbitrary shape, good for noisy data.

Math (K-Means example):

  1. Choose k cluster centers randomly.

  2. Assign each point to the nearest center (using Euclidean distance).

  3. Update centers as the mean of assigned points.

  4. Repeat until convergence.

2. Dimensionality Reduction

  • Purpose: Reduce the number of features while keeping essential information.

  • Examples:

    • Election system: Simplify voter data with hundreds of features into 2D for visualization.

    • Image compression.

    • Speeding up training for supervised models.

  • Algorithms:

    • PCA (Principal Component Analysis): Projects data onto directions of maximum variance.

    • t-SNE: Preserves local structure for visualization.

    • Autoencoders: Neural networks that compress and reconstruct data.

Math (PCA example):

  1. Compute covariance matrix of data.

  2. Find eigenvectors (principal components).

  3. Project data onto top components.

3. Association Rule Learning

  • Purpose: Discover relationships between variables.

  • Examples:

    • Election system: Find patterns like “Voters who register late often vote online.”

    • Retail: Market basket analysis (items bought together).

  • Algorithms:

    • Apriori: Finds frequent item sets and builds rules.

    • Eclat: Uses depth-first search for efficiency.

Rule Example: If {Milk, Bread} → {Butter}, confidence = 80% (means 80% of people who buy milk and bread also buy butter).

4. Anomaly Detection

  • Purpose: Identify unusual patterns.

  • Examples:

    • Election system: Detect fraudulent votes without needing labeled fraud data.

    • Finance: Spot unusual transactions.

    • Cybersecurity: Detect intrusions.

  • Algorithms:

    • Isolation Forest: Randomly partitions data; anomalies are easier to isolate.

    • One-Class SVM: Learns boundary around normal data, flags outliers.

K-means clustering for voter segmentation

You’re looking to uncover natural groups in your voter data without labels. K-means is a fast, practical choice to segment voters by behavior, demographics, or engagement patterns for smarter resource planning and fraud screening.

Concept and objective

  • Core idea: Partition data into k groups so each point is assigned to the nearest cluster center.

  • Output: Cluster assignments and centroids you can interpret (e.g., “young, infrequent voters”, “consistent early voters”).

  • Use cases in elections:

    • Turnout planning: Allocate resources to clusters likely to form crowds.

    • Engagement strategy: Tailor communication per cluster.

    • Pre-fraud screening: Identify atypical subgroups for deeper checks.

Mathematics and algorithm

  • Objective function:

minimize J=∑i=1k∑x∈Ci∥x−μi∥2

where Ci is cluster i and μi its centroid.

  • Distance metric:

d(x,μi)=∥x−μi∥2

(Euclidean distance; other metrics are possible but standard K-means uses this.)

  • Algorithm (Lloyd’s method):

    1. Initialize centroids (k-means++ preferred for stability).

    2. Assign points to the nearest centroid.

    3. Update centroids as the mean of assigned points:

μi=1∣Ci∣∑x∈Cix

  1. Repeat steps 2–3 until assignments stop changing or improvement in J is tiny.

Choosing k and evaluating clusters

  • Elbow method (inertia): Plot total within-cluster sum of squares vs. k. Look for the “elbow” where improvement flattens.

  • Silhouette score: Measures how well points fit their cluster vs. neighboring clusters. Values near 1 are better; below 0.3 suggests weak structure.

s=b−amax⁡(a,b)

where a is mean intra-cluster distance and b is mean nearest-cluster distance.

  • Practical tip: Start with k∈[2,6]. Pick the smallest k that gives a high silhouette and interpretable segments.

Data preparation and feature design

  • Feature scaling:

    • Why: Euclidean distance is sensitive to scale.

    • How: Standardize features to zero mean, unit variance.

  • Feature selection (examples):

    • Behavioral: turnout_frequency, average_vote_time, online_vote_ratio.

    • Demographic: age, location_code (one-hot or target-encoded).

    • Engagement: times_app_opened, notifications_clicked.

  • Handling categorical variables: One-hot encode (beware high dimensionality). Consider dimensionality reduction if many categories.

Training script

python
import os
import numpy as np
import pandas as pd
from dotenv import load_dotenv
from sqlalchemy import create_engine
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

load_dotenv()
PG_URI = os.getenv("PG_URI")
MODEL_VERSION = os.getenv("MODEL_VERSION")
K_MIN = int(os.getenv("K_MIN", 2))
K_MAX = int(os.getenv("K_MAX", 7))
RANDOM_STATE = int(os.getenv("RANDOM_STATE", 42))

engine = create_engine(PG_URI)

# 1) Load features
features = [
    "age",
    "turnout_frequency",
    "avg_vote_hour",
    "online_vote_ratio",
    "times_app_opened"
]
df = pd.read_sql(f"SELECT voter_id, {', '.join(features)} FROM voters", engine)

# 2) Impute + scale
X = df[features].copy()
imputer = SimpleImputer(strategy="median")
X_imp = imputer.fit_transform(X)

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_imp)

# 3) Choose k via silhouette
best_k, best_sil, best_labels, best_model = None, -1.0, None, None
for k in range(K_MIN, K_MAX + 1):
    km = KMeans(n_clusters=k, init="k-means++", n_init=25, random_state=RANDOM_STATE)
    labels = km.fit_predict(X_scaled)
    sil = silhouette_score(X_scaled, labels)
    if sil > best_sil:
        best_k, best_sil, best_labels, best_model = k, sil, labels, km

print(f"Best k={best_k} silhouette={best_sil:.3f}")

# 4) Compute distance to center per voter (optional)
centers = best_model.cluster_centers_
dists = np.zeros(len(df))
for i, (x, c) in enumerate(zip(X_scaled, best_labels)):
    dists[i] = np.linalg.norm(x - centers[c])

# 5) Save assignments to voter_clusters
out = pd.DataFrame({
    "voter_id": df["voter_id"],
    "model_version": MODEL_VERSION,
    "cluster_label": best_labels,
    "distance_to_center": dists
})

with engine.begin() as conn:
    out.to_sql("voter_clusters", conn, if_exists="append", index=False)

# 6) Save centroids (inverse transform to original units for interpretation)
centroids_orig = scaler.inverse_transform(centers)
centroids_df = pd.DataFrame(centroids_orig, columns=features)
centroids_df.insert(0, "model_version", MODEL_VERSION)
centroids_df.insert(1, "cluster_label", range(best_k))

with engine.begin() as conn:
    centroids_df.to_sql("cluster_centroids", conn, if_exists="append", index=False)

print("Saved clusters and centroids.")
